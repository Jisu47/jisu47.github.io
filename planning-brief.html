왜 결과물이 아직 없는가

구현 전 검토 단계

현재는 기획 완성도와 우선순위를 고정하는 단계입니다. 구현 전에 문제 정의, 사용자 흐름, MVP 범위를 먼저 공개하면 결과물이 없어도 맥락 전달이 가능합니다.

  • 1. 요구사항 고정

    기획서 기준 필수 기능과 제외 범위를 분리해 MVP를 확정합니다.

  • 2. 화면 초안 또는 흐름 정리

    와이어프레임, 사용자 흐름도, 초기 정보 구조를 정리해 구현 리스크를 줄입니다.

  • 3. 프로토타입 또는 저장소 공개

    데모가 생기면 이 페이지의 상단 링크에 라이브 링크와 저장소를 추가합니다.

문서형 원문 연결

현재는 실제 PDF 원문이 연결되어 있고, 문서형 HTML 페이지는 웹에서 읽기 쉬운 보조 설명 페이지로 유지합니다. 필요하면 PDF 파일만 교체해도 같은 구조를 계속 사용할 수 있습니다.

PDF 원문 열기 문서형 HTML 열기 메인 프로젝트 카드로 돌아가기
  • 프로젝트 상태를 반드시 `기획 단계`로 표시하기
  • 아직 없는 데모나 저장소 링크는 만들지 않기
  • 문제 정의와 다음 단계를 한 화면에서 읽히게 유지하기
  • 원문 링크는 현재 PDF 경로를 기준으로 유지하고 파일만 교체하기
